# Form Inputs

## Purpose
Provides accessible text, select, radio, checkbox, and textarea controls with validation support.

## Accessibility Notes
- Every control has a visible label
- Required fields expose error messages after submit
- Hints and errors are associated through `aria-describedby`
- Invalid controls expose `aria-invalid`
- Radio buttons are grouped with `fieldset` and `legend`

## WCAG Criteria
- 1.3.1 Info and Relationships
- 2.1.1 Keyboard
- 2.4.3 Focus Order
- 3.3.1 Error Identification
- 3.3.2 Labels or Instructions
- 4.1.2 Name, Role, Value
